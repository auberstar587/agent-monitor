# Agent Monitor + 开会展示系统

## 项目目标
1. 监控OpenClaw Agent状态
2. "开会"展示功能 - 形象展现团队讨论过程

## 系统架构

```
┌─────────────────────────────────────────┐
│           前端展示层 (Web)               │
│  - Agent状态面板                         │
│  - 开会过程可视化（时间线/消息流）        │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│           后端服务层                     │
│  - Agent状态收集器                      │
│  - 会议记录器                           │
│  - WebSocket实时推送                    │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│           OpenClaw API                  │
│  - sessions_list                         │
│  - sessions_history                      │
│  - subagents                            │
└─────────────────────────────────────────┘
```

## 团队角色

| Agent | 角色 | 职责 |
|-------|------|------|
| canmou | 参谋 | 调研、分析 |
| creator | 笔杆子 | 内容生成 |
| yunying | 运营官 | 执行、监控 |
| evolver | 进化官 | 技能优化 |

## 工作流

### 开会模式
1. 提出议题 → canmou调研分析
2. 方案讨论 → 多人协作
3. 决策输出 → creator记录
4. 执行跟踪 → yunying监控
5. 复盘优化 → evolver技能提升

## 技术选型

- 前端：轻量HTML+CSS+JS
- 后端：Node.js (轻量)
- 实时：WebSocket
- 数据源：OpenClaw API
