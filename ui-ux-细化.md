### 1.2 工位模式 vs 会议室模式

| 维度 | 工位模式 | 会议室模式 |
|------|----------|------------|
| 布局 | 横向平铺，5个角色均匀分布 | 圆形/椭圆会议桌，角色围坐 |
| 间距 | 角色间距 40px | 角色间距围绕圆周均匀分布 |
| 气泡位置 | 角色上方 20px 处 | 气泡浮于角色附近，统一偏向上方 |
| 背景 | 深色桌面 (#1a1a2e)，轻微网格纹理 | 会议桌背景 (#16213e)，暖色台灯效果 |
| 视角 | 平视角度 | 略微俯视角度 (15°) |

### 1.3 大屏展示优化

```
分辨率适配策略:
- 1920x1080 (标准大屏): 角色卡片 280x320px，间距 60px
- 2560x1440 (2K大屏): 角色卡片 320x380px，间距 80px
- 3840x2160 (4K大屏): 角色卡片 400x480px，间距 100px

自动缩放: 使用 CSS transform: scale() 整体缩放
字号基准: 1080p下 14px，4K下自动 ×1.5

大屏专属优化:
- 隐藏底部控制栏（鼠标悬停时显示）
- 角色卡片尺寸增大 30%
- 气泡字号增大 20%
- 增加粒子背景效果
```

### 2.2 形象渲染形式

**推荐方案: SVG + CSS 动画**

```
技术选型理由:
- SVG: 矢量图形，缩放不失真，文件体积小
- CSS: 状态切换流畅，可利用 GPU 加速
- 组合: SVG 定义形状，CSS 控制动画

备选方案对比:
- Canvas: 性能好但开发成本高，适合大量粒子
- CSS 纯绘制: 简单但复杂造型困难
- WebGL: 性能最强，但开发复杂度过高
```

### 2.3 角色状态视觉设计

每种状态包含: 基础层 + 状态层 + 特效层

#### Idle (空闲/待命)
```
视觉表现:
- 身体微微上下浮动 (translateY: 0 ~ -5px)
- 浮动周期: 3s，ease-in-out，无限循环
- 眼睛: 正常睁开，偶发眨眼 (每5-8秒一次)

配色: 正常饱和度
光晕: 无
```

#### Working (工作中)
```
视觉表现:
- 身体前倾姿势 (-5deg rotate)
- 手部/工具轻微抖动
- 眼睛: 专注眯眼
- 周围: 小型齿轮/代码符号旋转

配色: 饱和度 +10%
光晕: 角色底部 10px 蓝色微光 (#4A90D9, opacity 0.3)
```

#### Speaking (发言中)
```
视觉表现:
- 嘴巴张开动画 (SVG path 变形)
- 身体轻微前倾并左右摇摆 (±3deg)
- 头部微抬
- 周围: 声波纹扩散效果 (3层同心圆)

配色: 饱和度 +20%，边缘发光
光晕: 角色周围 20px 彩色光晕 (角色主题色, opacity 0.4)
动画时长: 声波 1.5s 一周期，逐渐扩大消失
```

#### Meeting (会议中/倾听)
```
视觉表现:
- 坐姿/站姿保持端正
- 眼睛: 正常或略眯
- 耳朵图标闪烁 (表示在听)
- 偶尔点头动作 (每10-15秒)

配色: 正常饱和度
光晕: 无
辅助: 头顶小型问号/灯泡图标
```

### 2.4 状态切换动画

```
切换时长: 300ms
缓动函数: cubic-bezier(0.4, 0, 0.2, 1)

过渡效果:
- 颜色变化: 300ms 平滑过渡
- 位置移动: 400ms，带轻微弹性
- 光晕出现: 200ms fade-in，300ms fade-out
```

## 3. 动画效果规范

### 3.1 工位 → 会议室 移动动画

```
触发: 点击"开始会议"按钮
动画类型: 飘移 + 淡入 + 变形

序列:
1. T+0ms:     工位角色原地 scale(1) → scale(0.8)，opacity 1 → 0.5
2. T+200ms:  角色向上飘起 (translateY: 0 → -50px)，同时向目标位置水平移动
3. T+800ms:  角色到达会议桌对应位置
4. T+1000ms: scale 0.8 → 1，opacity 0.5 → 1
5. T+1200ms: 会议桌背景淡入 (opacity 0 → 1)

总时长: 1200ms
缓动: cubic-bezier(0.34, 1.56, 0.64, 1) // 轻微弹性

路径: 直线移动，可选贝塞尔曲线路径
```

### 3.2 发言动画

```
气泡弹出:
- 延迟: 发言开始后 100ms
- 时长: 300ms
- 效果: scale(0) → scale(1)，opacity 0 → 1
- 缓动: cubic-bezier(0.34, 1.56, 0.64, 1)

角色动作:
- 嘴巴: SVG 变形动画，200ms
- 身体: 左右摇摆 ±3deg，周期 500ms
- 声波: 3层圆环，从 20px 扩大到 80px，opacity 1 → 0

打字效果 (可选):
- 每字符 50ms 延迟显示
- 光标闪烁 530ms 周期的 blink 动画
```

### 3.3 多角色同时发言

```
优先级策略:
- 最多同时显示 2 个大气泡
- 第 3 个及以后的发言进入"队列"
- 队列中的发言显示小图标提示

视觉处理:
- 主发言: 气泡正常大小 (宽度 300px)，完整动画
- 次发言: 气泡缩小 70%，位置偏移避免重叠
- 队列提示: 角色头顶显示数字角标 (1, 2, 3...)

冲突解决:
- 同一角色连续发言: 合并为一个气泡，新内容追加
- 不同角色同时发言: 左右分布，靠近各自角色
```

## 4. 气泡 UI 设计

### 4.1 气泡样式

```
基础样式:
- 背景色: #FFFFFF (白色)
- 边框: 2px solid [角色主题色]
- 圆角: 16px (大圆角)，8px (小圆角)
- 内边距: 16px 20px
- 最大宽度: 320px
- 最小宽度: 120px
- 阴影: 0 4px 20px rgba(0,0,0,0.15)

尾巴样式:
- 位置: 气泡下方，距离角色头顶 8px
- 尾巴宽度: 12px，高度: 8px
- 颜色: 与气泡背景色一致 (白色)
- CSS 实现: ::before + ::after 伪元素

字体:
- 字号: 14px (对话)，12px (元信息)
- 行高: 1.5
- 颜色: #333333 (主文字)
- 对话人名称: 角色主题色，12px，加粗
```

### 4.2 气泡位置规则

```
工位模式:
- 位置: 角色上方 20px 居中
- 气泡尾巴向下指向角色
- 多气泡时垂直堆叠，间距 8px

会议室模式:
- 位置: 角色前方的会议桌区域
- 所有气泡统一向上飘起 (解决下方空间不足)
- 气泡尾巴向下

边界处理:
- 超出屏幕边界: 自动调整位置，确保在可视区域内
- 左右边界: 气泡自动靠左或靠右，尾巴跟随调整
```

### 4.3 刷屏处理

```
消息限流:
- 同一角色连续消息: 500ms 内合并显示
- 超出 5 条未显示: 显示 "...还有 X 条消息" 折叠按钮

折叠展开:
- 点击展开: 显示全部消息，向上滚动
- 展开后自动滚动到最新
- 最大展开高度: 屏幕 40%

历史消息:
- 进入会议时加载最近 20 条
- 滚动加载更多 (每页 20 条)
- 新消息自动追加，页面保持当前位置
```

## 6. 技术选型建议

### 6.1 推荐技术栈

```
主框架: React 18 + TypeScript
├── 理由: 组件化开发，状态管理成熟，生态丰富

动画库: Framer Motion
├── 理由: React 官方推荐，性能好，API 简洁
├── 替代: GSAP (更强大但学习曲线陡)

样式方案: Tailwind CSS + CSS Modules
├── 理由: 快速开发，响应式方便
├── 自定义: CSS Variables 管理主题色

图标: Lucide React
├── 理由: 轻量，风格统一

状态管理: Zustand
├── 理由: 轻量级，适合中型状态管理

构建工具: Vite
├── 理由: 极速启动，热更新快
```

### 6.2 动画性能保证

```
策略:
1. GPU 加速: 使用 transform 和 opacity 动画
   - ✅ transform: translate(), scale(), rotate()
   - ✅ opacity
   - ❌ 避免: width, height, margin, padding

2. will-change 属性:
   - 对动画元素添加 will-change: transform, opacity
   - 动画结束后移除

3. 减少重绘:
   - 使用 CSS containment
   - 避免布局抖动

4. 虚拟滚动:
   - 消息列表使用 react-window
   - 角色列表虚拟化 (超过 10 个时)

5. requestAnimationFrame:
   - 自定义动画使用 RAF 而非 setInterval
```

### 6.3 响应式实现

```
断点:
- mobile: < 768px (手机)
- tablet: 768px - 1024px (平板)
- desktop: 1024px - 1920px (标准屏)
- large: > 1920px (大屏/电视)

实现方式:
1. Tailwind 响应式前缀
   - sm:, md:, lg:, xl:, 2xl:

2. 容器查询 (Container Queries)
   - 组件自适应容器而非视口

3. Rem 相对单位
   - 字号: clamp(14px, 2vw, 20px)
   - 间距: 使用 CSS 变量统一管理

4. 大屏特殊处理
   - 检测 > 1920px 时启用"大屏模式"
   - 整体 scale 缩放适配
```
